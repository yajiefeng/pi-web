# New defaults to Herdr

Pi-web's `+ New` action will default to creating a Herdr-owned session, even before Herdr-owned sessions can receive web chat input. After `+ New`, pi-web immediately shows a Pending Herdr Session while it waits for Herdr to report a session binding, then opens the linked Herdr-owned session. The creation API returns the Herdr agent id/label but does not promise a session id; runtime-status SSE is responsible for discovering the eventual session binding by watching that exact agent id. Pi-web does not infer the new session from cwd, timestamps, or the latest session file. Pi-web-created Herdr agents use names like `pi-web-<short-project-name>-<short-random>` and start with `herdr agent start <name> --cwd <cwd> -- pi`, letting Herdr own the pane cwd while pi-web avoids coupling to extra Pi CLI flags. If creation fails or no binding appears within 15 seconds, pi-web shows explicit `Focus Herdr agent` when available, `Try Herdr again`, and `Create web session instead` actions rather than silently falling back or automatically closing the Herdr pane. Even when Herdr is unavailable before creation starts, the user must explicitly choose the pi-web-managed fallback. This moves the product toward Herdr as the default runtime owner while preserving an escape hatch for Herdr downtime.
